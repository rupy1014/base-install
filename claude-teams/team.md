# /team

에이전트 팀 생성 명령어

## 설명
Agent Team을 생성하고 작업을 분배합니다.
사용자의 작업 설명을 분석하여 적절한 팀 구성을 자동으로 결정합니다.

## 사용법
`/team [작업 설명]`

## 팀 생성 프로세스
1. **작업 분석**: 병렬 처리 가능한 단위로 분해
2. **팀 구성**: 작업 특성에 맞는 teammate 역할 배정. 각 팀원은 `sonnet[1m]` (1M 컨텍스트 윈도우) 모델을 사용하여 생성한다.
3. **태스크 분배**: 공유 태스크 리스트에 작업 등록 및 할당
4. **실행 및 조율**: 팀원 간 진행상황 조율, 결과 종합

## 팀 구성 예시

### 코드 리뷰
3명: 보안 / 성능 / 테스트 커버리지

### 기능 개발
3명: 프론트엔드 / 백엔드 / 테스트

### 디버깅
경쟁 가설 방식 - 각 팀원이 다른 가설을 검증하며 서로 반박

## 모델 설정

팀원은 `sonnet[1m]` 모델(1백만 토큰 컨텍스트 윈도우)을 기본으로 사용한다.
팀 생성 시 리드가 `/model sonnet[1m]`을 설정하면 팀원이 이를 상속한다.

| 모델 | 용도 | 컨텍스트 |
|------|------|----------|
| `sonnet[1m]` | 팀 작업 기본 (권장) | 1M 토큰 |
| `sonnet` | 일반 코딩 | 200K 토큰 |
| `opus` | 복잡한 아키텍처 결정 | 200K 토큰 |
| `haiku` | 빠른 탐색/검색 | 200K 토큰 |

## 키보드 단축키
- **Shift+Up/Down**: 팀원 간 전환 및 직접 메시지
- **Shift+Tab**: delegate 모드 토글 (리드는 조율만)
- **Ctrl+T**: 태스크 리스트 토글
- **Enter**: 팀원 세션 진입 / **Escape**: 현재 턴 중단

## 팀 종료
작업 완료 후 "Clean up the team" 입력

## 주의사항
- 같은 파일을 여러 팀원이 수정하지 않도록 작업 분리 필요
- 팀원당 5-6개 태스크가 적절
- 토큰 사용량이 일반 세션보다 크게 증가
- `sonnet[1m]` 사용 시 200K 토큰 초과분에 프리미엄 요금 적용 (입력 2배, 출력 1.5배)
